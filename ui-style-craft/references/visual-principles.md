# 视觉原则

给“从零构建”场景使用。每条都可量化、可检查，不是模糊建议。

## 1. 层级（Hierarchy）

用**字重 + 对比度 + 间距**三件套控制层级，而非单纯放大字号。

- 主标题：字重 `semibold`(600) + 字间距 `tighter`(-0.02em) + 与下方内容间距 ≥ `--space-6`(24px)。
- 次级信息：降一级字阶 + `text-secondary` 颜色，而非换色或加粗。
- 禁用：超大 H1(>4rem) 硬撑层级、用高饱和色制造“重点”。

## 2. 留白（Whitespace / Density）

密度分级对应的间距倍率（见 `space` 的 `--density`）：

| 密度 | 倍率 | 适用 |
| --- | --- | --- |
| compact | 0.75 | 数据密集型仪表盘、表格 |
| normal | 1.0 | 通用后台 / 应用 |
| comfortable | 1.5 | 营销页、阅读 / 文档 |

- 卡片内距：normal 下 24–32px（`--space-6` ~ `--space-8`）。
- 区块之间垂直间距走节奏分级（见第 4 节），不随手给 13px / 17px。

## 3. 对齐（Alignment）

- 栅格基准 **4px**：所有尺寸为 4 的倍数（size 阶梯已保证）。
- 同一组的元素左对齐到统一基线；避免“既要居中又要左对齐”的混排。
- 文本列最大宽度约 70ch，页面容器 ≤ 1200px（`--space-16` 以上再做外边距约束）。

## 4. 节奏（Rhythm）

模块间垂直节奏走分级，而非均匀随机：

- 组内元素间距：`--space-2`(8) / `--space-3`(12) / `--space-4`(16)
- 组件间距：`--space-6`(24) / `--space-8`(32)
- 区块间距：`--space-10`(40) / `--space-12`(48) / `--space-16`(64)

相邻层级至少差一级，制造可感知的 grouping。

## 5. 纵深（Depth）

三层投影体系，着色到背景色相（`shadow` 的 tinted 值），近→远：

- 近（sm）：按钮、标签、输入框聚焦——轻 1px 投影。
- 中（md）：卡片、下拉、弹出——分层双投影。
- 远（lg/xl）：模态、悬浮面板、抽屉——更柔更散。

- 仅当投影能传达“层级”时才用卡片；不要给每个块都加卡片。
- 禁用：默认纯黑阴影、霓虹外发光、投影不随层级分级。

## 6. 一致性检查清单

- [ ] 所有颜色来自 `color.semantic` / `color.neutral`，无散落 hex。
- [ ] 所有间距来自 `space` 阶梯。
- [ ] 字号来自 `font.size` 字阶；行高来自 `lineHeight`。
- [ ] 圆角来自 `radius`；阴影来自 `shadow`（tinted）。
- [ ] 对比度经 `validate-contrast.mjs` 全 PASS。
